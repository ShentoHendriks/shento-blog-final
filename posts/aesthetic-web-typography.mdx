## Modern `reset.css`
At the start of your project, you should always use a `reset.css` to make sure your website looks the same in every browser. It removes default styles and gives you a clean base to work from.

<Dos dos>**Load `reset.css` first.** This ensures that your own styles don’t get overwritten or affected by inconsistent default browser styles.</Dos>

<Callout>
If you use Tailwind CSS, you have a built-in `reset.css` called '[Preflight](https://tailwindcss.com/docs/preflight)'.
</Callout>


### Josh Comeau's `reset.css`
When it comes to `reset.css`, my personal favorite is [Josh Comeau's reset.css](https://www.joshwcomeau.com/css/custom-css-reset/). It’s **modern**, thoughtfully crafted, and incorporates sensible **best practices** that align well with today’s web development needs.

<Dos dos>It uses a more **intuitive box-sizing** model `box-sizing: border-box`</Dos>
<Dos dos>Accessible line-heights</Dos>
<Dos dos>Improved text rendering with `-webkit-font-smoothing: antialiased`</Dos>
<Dos dos>Much more ...</Dos>

### Eric Meyer's `reset.css`
On the other hand, the widely known [Eric Meyer's reset.css](https://meyerweb.com/eric/tools/css/reset/) is more old-school and, quite frankly, a bit too aggressive for most modern projects.

<Dos donts>Doesn't come with the `box-sizing: border-box`</Dos>
<Dos donts>Doesn’t attempt to fix **inconsistencies** between browsers—just erases everything </Dos>
<Dos donts>**Not opinionated enough** in my opinion</Dos>

## Fixing inconsistent font sizes
Without a systematic approach to font sizing, your typography becomes inconsistent and chaotic. You'll find yourself randomly picking sizes—making `H2` headings `22px`, then `24px`, then `20px`—creating visual discord that feels unprofessional and hard to scan.

### Typescales

A **typescale** solves this by creating consistent relationships between all your text elements.

Start with your **base font size**: `16px` or `18px`.

From this foundation, use mathematical ratios to generate harmonious size typography.
Popular typescale ratios include:

- **1.125 (Major Second)** - subtle, conservative
- **1.25 (Major Third)** - balanced, versatile
- **1.414 (Augmented Fourth)** - dynamic, bold

### How does this work?
You start with a base font (like 16px), then **multiply by the ratio** to get each larger size, or **divide by the ratio** to get smaller sizes.

<Callout type="info">
Tools like [typescale.com](https://typescale.com) can generate your complete typescale automatically. While also giving a preview.
</Callout>

## Consistent Vertical Spacing

**Vertical spacing** is all about the empty space above and below your text. This includes the space between lines in a paragraph (called `line-height` by web developers and leading by designers) and the space (margin) between sections and elements (such as images, paragraphs) 

When vertical spacing isn't done well, it makes **reading much harder**. Text that's too cramped feels overwhelming and difficult to follow. Text with too much space feels choppy and disconnected, like the ideas don't belong together.

Good vertical spacing makes your content easier to read and more professional. It helps readers move naturally from one idea to the next and makes your text look clean and well-organized.

### Line Height Guidelines
For **line-height**, the rule is: **the smaller the text, the more line-height is needed**. An example you can follow is:
- **Body text**: 1.4–1.6 (times the **base** font)
- **Large headings (H1-H2)**: 1–1.2
- **Smaller headings (H3-H6)**: 1.2–1.4 

All those values **aren't** set in stone. **Experiment** and find the best values for yourself!

### Fixing Margin Collapse
Top and bottom margins of elements are sometimes **collapsed** into a single margin that is equal to the **largest** of the two margins. We can fix this weird behavior by doing this:

```css
.example {
  display: flex;
  flex-direction: column;
}
```
Now we have more **fine-tuned control** how margins work between elements.

### Vertical Margin Guidelines

<Callout type="warning">
This following advice is only suitable for **content-heavy typography** like **blog posts**.
</Callout>

The spacing between elements is quite complex, but here are a few **dos** and **donts**. You can **fine-tune** them as you want, but it gives you a general idea of the applied principles:

<Dos dos>Headings must have **more** `margin-top` than `margin-bottom`</Dos>
<Dos dos>All other elements's must **only** have a `margin-bottom`</Dos>
<Dos dos>The values of those `margin-bottom` should be consistent. Preferably either a reused **constant**, or values multiplied by 4</Dos>
<Dos dos>The `margin-bottom` of a `li` must be at least the **half** of the `ul` and `ol`</Dos>
<Dos donts>Avoid `margin-bottom` on the **the last element** in a list</Dos>
<Dos donts>Avoid `margin-top` on the first element **after** a header </Dos>
<Dos donts>Avoid `margin-top` on the the **first** element inside a **container** (like a page, section, or card) `margin-bottom` on the **last** element. This prevents redundant spacing at the edges.</Dos>
<Dos donts>Avoid `margin-bottom` on all elements **before** a heading</Dos>

:root {
  --margin-base: 4px;
}

.container {
  max-width: 800px;
  width: 100%;
  margin: 0 auto;
  padding: 1em;

  display: flex;
  flex-direction: column;

.prose li {

## Conclusion

Good typography isn't about rigid rules—it's about creating systems that work consistently across your project. By establishing a CSS reset, implementing a mathematical typescale, and applying thoughtful vertical spacing, you've built a strong foundation that eliminates guesswork and creates visual harmony.
